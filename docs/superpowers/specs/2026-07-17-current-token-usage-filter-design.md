# 按当前令牌过滤用量 + 首页模型区限高

**日期：** 2026-07-17  
**状态：** 已批准  

## 问题

1. 首页按账户汇总所有令牌的模型用量，两条（或多条）令牌的模型混在一起。
2. 模型行过多时挤占空间，底部 7 日柱状图被挡住。

## 目标

- Keys 页「设为当前」后，首页模型列表与 7 日柱状图只统计该令牌。
- 模型区可视高度约 2 行，超出可滚动，柱状图始终可见。
- 无当前令牌时仍显示账户全部用量。
- 账户余额仍为整户余额，不按令牌拆分。

## 方案

采用 New API `/api/log/self` 的 `token_name` 查询参数，用当前令牌的 `name` 过滤日志后再聚合。

### 后端

- `fetch_log_self` 增加可选 `token_name`；非空时写入 query（需 URL 编码）。
- `fetch_usage_summary`：若存在 `current_key_id` 对应 key，则用其 `name` 过滤。
- 响应增加可选 `filterTokenName`，供前端展示当前过滤令牌名。

### 前端

- 模型区：去掉硬截断为 4；改为全部模型 + `max-height` 约 2 行 + `overflow-y: auto`。
- 有 `filterTokenName` 时在模型区上方显示「当前令牌：xxx」。
- 今日 Token / 7 日合计 / 柱状图均使用已过滤的 `usage` 数据。

### 风险与兜底

- 手动添加的令牌若名称与面板 `token_name` 不一致，过滤结果可能为空；同步来的令牌一般一致。
- 日志仍受 `page_size` 限制；按令牌过滤后压力更小，分页增强可后续再做。

## 非目标

- 首页临时令牌下拉（不改 current）。
- 按令牌拆分账户余额。
- 修改 Keys 页同步逻辑。
